# Helm (web edition)

This is a Vue 3 / Vite port of the Helm Electron app, meant to live at
`app-src/` alongside your marketing site's root `index.html` and a
`Try it in your browser` button pointing at `./app/`.

## Repo layout

```
index.html       ← your existing landing page (unchanged, except the new CTA)
app-src/          ← this Vue app's source — what you edit/develop in
app/              ← generated by `npm run build` — what actually ships
```

`app-src/vite.config.js` sets `build.outDir` to `../app`, so running the
build from inside `app-src/` drops the compiled app straight into `app/`
next to your landing page — no manual copying needed. Deploy the whole
repo root (landing page + `app/`) as a static site, and the "Try it in
your browser" button on the landing page will just work via `./app/`.

The UI, routing, Pinia stores, charts, and all financial logic are
unchanged from the desktop app — only the persistence layer is different.


## What's different from the Electron app

The Electron app silently wrote `data.json` into the OS's app-config
folder (`app.getPath('userData')`) on every change, and loaded it
automatically on launch. This web build does **not** do that. Instead:

- **Create** — start a brand-new, empty data file from the welcome screen
  (`src/components/DataGate.vue`).
- **Import** — load a previously exported `data.json` / backup file
  (drag-and-drop or file picker), also from the welcome screen, or later
  from Settings → Restore.
- **Export** — download the current data as `data.json` at any time from
  Settings → Backup.

While the app is open, your working copy lives in memory and is mirrored
into the browser's `localStorage` purely so an accidental refresh doesn't
lose your work mid-session — that's a convenience cache, not the "real"
save. The `data.json` file you export is the actual durable copy; treat
it the same way you would the Electron app's backup file. Settings →
"Clear data & start over" wipes the browser copy entirely and returns to
the create/import screen.

## Architecture notes

- `src/lib/dataEngine.js` — a faithful port of `electron/store.cjs`
  (same record shapes, same migrations, same id-assignment logic), just
  swapping `fs`/`app.getPath` for `localStorage`.
- `src/lib/csvImport.js` — a straight port of `electron/csvImport.cjs`.
- `src/lib/webApi.js` — implements the same `window.api.*` surface that
  `electron/preload.cjs` exposed via IPC, so every Pinia store and
  component works completely unchanged. CSV import and backup
  export/import use a real `<input type="file">` picker and a `Blob`
  download instead of Electron's native dialogs.
- `src/components/DataGate.vue` + the `needsDataGate` flag in `App.vue`
  reproduce "first run" — except the user picks Create or Import instead
  of it happening silently.

## Running it

```bash
cd app-src
npm install
npm run dev      # http://localhost:5173 — dev server, hot reload
npm run build     # builds straight into ../app (next to the landing page)
npm run test      # unit tests for utils/ (unchanged from the desktop app)
```

After building, open the repo root's `index.html` (e.g. via `npx serve .`
from the repo root) and click "Try it in your browser" — it loads
`./app/index.html`.

There is no `electron/` folder, no `electron-builder` config, and no
`dist:*`/`electron:*` scripts — `app/` is a pure static site you can
deploy anywhere (Netlify, GitHub Pages, S3, etc.) alongside the landing
page.
